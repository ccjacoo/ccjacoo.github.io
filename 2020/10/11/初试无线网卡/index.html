    初试无线网卡

    3. 总结：

    这学期要上无线网络安全课程，所以最近鼓捣了一个无线网卡来玩玩。

    我们选择的网卡必须要能被kail识别，且能支持水滴，cd linux，普通的网卡肯定是不行的。而且大部分的网卡是普遍具有监听功能的，但是能 injection封包注入的网卡那是少之又少。我在网上查了很多资料，最后听取了大牛的意见，选择了 T L - W N 7 2 2 N 这款网卡（注意一定要认准 v1 版本，v2和v3版本不支持kali哦，一定要注意）。(听说用的人挺多的，后来自己实测，针不戳，一切操作都很顺利)

    今天中午网卡终于到了，是二手的（usb没有盖子呜呜呜），从吉林那儿邮过来真不容易啊

    初次认识usb无线网卡：

    拿到网卡后就迫不及待拿来玩了。

    插上网卡， 打开kali虚拟机，选择连接网卡，示意图如下:

    这是一个usb外联设备，所以可以选择连接或断开 ，一旦连接，原先插在物理机上的网卡就犹如插在虚拟机实机上一样。

    输入ifconfig 查看所有可识别的网卡：

    我这个网卡的芯片是 Atheros ，是USB2.0

    无线网卡一共有四个模式(mode)，分别是 Managed 、 Master 、 Ad hoc 、 Monitor(监听)

    Managed 模式：该模式用于你的无线客户端直接与无线接入点（Access Point，即AP）进行接入连接。在这个模式中，我们可以进行无线接入internet上网，无线网卡的驱动程序依赖无线AP管理整个通信过程。

    Master 模式：一些无线网卡支持Master模式。这个模式允许无线网卡使用特制的驱动程序和软件工作，作为其他设备的无线AP。它主要使用于无线接入点AP提供无线接入服务以及路由功能。比如无线路由器就是工作在Master模式下了，对于普通的pc机来说，如果有合适的硬件它也可以变成一台无线AP。

    Ad hoc模式：当你的网络由互相直连的设备组成时，就使用这个模式。在这个模式中，各设备之间采用对等网络的方式进行连接，无线通信双方共同承担无线AP的职责。

    Monitor 模式：Monitor模式主要用于监控无线网络内部的流量，用于检查网络和排错。要使Wireshark捕获无线数据包，你的无线网卡和配套驱动程序必须支持监听模式（也叫RFMON模式）。

    我们进行无线渗透测试的时候需要用到监听模式，所以要转换为 monitor mode

    网卡转换为监听模式的命令是： airmon-ng start wlan0

    root@KALI:~# airmon-ng start wlan0

    PHY Interface Driver Chipset

    phy0 wlan0 ath9k_htc Qualcomm Atheros Communications AR9271 802.11n

    ​ (mac80211 monitor mode vif enabled for [phy0]wlan0 on [phy0]wlan0mon)
    ​ (mac80211 station mode vif disabled for [phy0]wlan0)

    发现它这里说，wlan0mon是可用的(enabled)，并给了个[phy0]，而原先的 wlan0不可以用了。

    再次 ifconfig 就会发现自动在原先 wlan0的后面加了mon ，变为了 wlan0mon，如下图：

    开启监听： airmon-ng start wlan0mon

    输入 iwconfig ，发现mode已经变为了Monitor，此时就可以开始监听啦。

    输入 airodump-ng wlan0mon ，扫描周围的AP目标。

    可以查看每个AP的MAC地址、信号强度(值越小信号越好)、信道、加密方式、WIFI名称等信息。

    做到这里的时候我没有监听抓包继续做下去了，于是想要关闭网卡，但是不知道该怎么关，后来自己鼓捣出来方法了。

    输入 airmon-ng stop wlan0mon ，然后显示：

    root@KALI:~# airmon-ng stop wlan0mon

    PHY Interface Driver Chipset

    phy0 wlan0mon ath9k_htc Qualcomm Atheros Communications AR9271 802.11n

    ​ (mac80211 station mode vif enabled on [phy0]wlan0)

    ​ (mac80211 monitor mode vif disabled for [phy0]wlan0mon)

    这里说的是 wlan0mon 已经不存在了，而 wlan0 依然存在(enabled)。

    但是我 ifconfig 并没有显示 wlan0 的信息，而我 输入 ifconfig wlan0依然能查到wlan0的信息，

    于是我接着输入 ifconfig wlan0 up，尝试开启wlan0网卡，于是再次ifconfig就又重新出现了wlan0，好耶。

    当然了，不用wlan0时就输入 ifconfig wlan0 down，关闭网卡， 然后虚拟机外联设备那里选择 ‘取消连接’ 即可，然后拔掉usb网卡。万事大吉。

    抓握手包&暴力破解密码：

    OK，第一部分初次认识usb网卡部分结束。接下来是尝试监听抓包，然后尝试破解密码。

    之前我做过一道ctf题目，给你一个wifi握手包，然后让你用 aircrack-ng工具去破解密码。字典我用的是kali自带的 rockyou字典，（字典位置： /usr/share/wordlists/rockyou.txt.gz）需要解压为txt再使用。（另外，aircrack-ng 有专门的命令来创建字典的。

    仍然输入如下命令扫描周围的AP目标：

    airodump-ng wlan0mon

    我们选择的目标是： essid为 6-202 的AP

    输入

    airodump-ng wlan0mon –bssid 8C:53:C3:32:88:F8 -c 7 -w wpa2

    抓取这个AP的数据包。

    第一行显示了AP的MAC地址，发出的beacon帧，以及接收的Data数据。随着时间推移，beacon帧会越来越多，Data数据也会增加。

    然后还有第二行，第一列是AP的MAC地址，第二列( STATION)是某个人的MAC地址，说明此时这个AP有用户连接。

    开始抓包后，我们需要等用户连接我们伪装的wifi，才能抓到包。需要利用aireplay-ng工具，给连接到wifi的一个设备发送deauth（反认证）包，使其断开wifi，随后它自然会再次连接wifi。这时我们及时抓取握手包，就达到了我们的目的了。

    所以接下来的操作有点意思，重新打开一个新终端（上一个正在查看wifi信息的命令终端一定不要关闭）
    然后在终端处输入：

    aireplay-ng -0 2 -a 8C:53:C3:32:88:F8 -c 18:AF:61:CA:58:62 wlan0mon

    参数解释：-0 表示攻击次数， -a表示目标AP， -c 表示要强制断开连接的设备 ，最后是指定网卡接口

    这条命令的意思是将该wifi下的所有设备踢下线 如果你想某种恶作剧(恶趣味)的话 也可以使用这条命令

    当这里出现了 WPA handshake 字眼的时候表明我们抓取握手包成功，捕获到了带有密码的数据包。

    此时的握手包默认存储在了 /root 下，我们直接在当前终端 ls 一下， 看见一个.cap后缀的数据包（我自己的是wpa2-01.cap），这个就是握手包了。

    注意！我们捕获到数据包之后 一定要切换到输入了攻击指令的命令终端 按下Ctrl+c 停止攻击（直接关闭掉这个终端也可）。如果不停止攻击的话 kali会不间断的对路由器进行断网操作 会使路由器瘫痪

    之后就是暴力破解密码的操作了。我在root目录下创建了名为wifi的文件夹，然后把cap数据包和字典都放进去了。为了实验方便，我直接在字典里输入正确的密码，百分百命中！耶

    命令：

    aircrack-ng -w /root/wifi/dir.txt wpa2-01.cap

    参数解释： -w表示字典位置 ，之后跟要破解的握手包

    OK，总之，如果字典够强大，则会显示出密码，破解成功！
