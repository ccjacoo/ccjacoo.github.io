    说明：

    本篇只是我从印象笔记里整理之后搬运过来的，都是我本人在学习过程中所记录下来的，所以可能会有点零散。若想要全面学习的话，请自行百度叭，网上不乏讲得细的文章。

    前提知识：

    • 直接获取栈上被视为第n+1个参数的数值 ： %n$x ，n对应着第n个输出参数，那相对于输出函数来说那就是第n+1个参数了。
    • %hhn 向某地址写入1字节
    • %hn 向某个地址写入2字节
    • %n 向某个地址写入4字节
    • %lln 向某地址写入8字节
    • %c ：输出字符，配上%n可用于向指定地址写数据
    • %d：输出十进制整数，配上%n可用于向指定地址写数据
    • %x：输出16进制数据，如%i$x表示要泄露偏移i处4字节长的16进制数据，%i$lx表示要泄露偏移i处8字节长的16进制数据，这一点32位和64位都一样。
    • %p：输出16进制数据，与%x基本一样，只是附加了前缀0x，32位下输出4字节，64位下输出8字节。
    • %s：输出字符串，将偏移处指针指向的字符串输出，如%i%s表示输出偏移i处地址所指向的字符串，一般用于读取GOT表信息，这一点32位和64位都一样
    • %n：将%n之前已经打印的字符个数赋值给偏移处指针所指向的地址位置，如%100x%10$n表示将0x64写入偏移10处保存的指针所指向的地址（4字节），%$hn表示写入的地址空间为2字节，%$hhn表示1字节，%$lln表示写入的地址空间为8字节，这一点32位和64位都一样
    • %n是关键要素，其他格式化参数可用于读取信息或配合%n写数据。

    漏洞存在点：

    printf函数真是个憨憨啊，它并不知道自己现在的参数个数有几个，但是它的内部却有个指针用来索检格式化字符串。对于遇到特定类型%，就去执行取相应参数的值，直到索检到格式化字符串结束。真是个憨憨。

    凭经验，如果printf语句没有带格式化字符参数的话，那么就一定存在格式化字符串漏洞。

    printf(s);

    漏洞的利用手段：

    1.搞破坏，使程序崩溃。因为%s对应的参数地址不合法的概率还是比较大的。所以直接输入无数个%s让其遇到不合法地址然后崩溃。

    2.泄露内存，泄露栈内存：获取某个变量的值；获取某个变量对应地址的内存。 泄露任意地址内存：利用got表得到libc函数地址，进而获取其他libc函数地址；盲目地dump整个程序，获取有用信息。

    3.任意地址写。

    计算格式化字符的偏移：

    运行程序，输入以下数据来得知格式化字符串在第几个参数上。

    AAAA%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p

    为了方便看清，每个参数之间都用点号分隔开。

    若得到的反馈是：

    AAAA0xffaab160.0xc2.0xf76146bb.0x41414141.0x70257025.0x70257025……

    看到我们输入的AAAA（0x41414141）出现在了输出内容的第5个位置上（算上开头的AAAA）。那么就知道格式化字符串的起始地址正好是输出函数的第5个参数，是格式化字符串的第4个参数， 即可以写作 %4$s 。这两个概念要弄清。

    知道了格式化字符在输出函数调用时是第K个参数，那么我们就可以通过

    addr%k$s

    来泄露指定addr的内容！
    如果在此基础上，我们把第四个参数对应的地址改为 scanf@got ，就可以泄露scanf的got地址。

    用gdb计算参数之间的偏移：

    假如栈中有个变量存放flag，其地址为0x7fffffffdda8， 那么gdb指令为：

    fmtarg 0x7fffffffdda8

    就可以计算我们输入的格式化字符串数据的地址与存flag的这个变量地址之间的偏移
    例如得到：

    The index of format argument : 10 (“%9$p”)

    控制符 %n 的利用：

    在格式化控制符中有一个 %n ，它用于把当前输出的所有数据的长度写回一个变量中去。

    由于可能会造成溢出漏洞从而进程被恶意代码劫持，现如今该控制符貌似很早之前就被弃用了。
    现在只有在 vc 6.0++ 和 linux 上还可以用。

    举一个栗子：

    #include<stdio.h>
int main(int argc, char const *argv[])
{
    int length=0;
    printf("hello%n\n",&length);
    printf("%d\n",length);
    return 0;
}
//输出：
//hello
//5

    该程序 首先会输出 hello ，然后把字符串长度5存回 &length变量里，第二次输出length变量的值即是5。

    任意地址写(32位)：

    假如我们的输入将放在变量C里，通过打印输出格式化字符串内容，知道了变量C存储在第6个参数处，我们需要借助printf函数的特殊格式化字符 %n 的特征来修改C的值，实现任意地址写任意值。

    [addr of C]%012d%6$n

    addr of C的长度为4字节，再输入12字符才可以达到16个字符，以便于修改C的值为16。
    payload表述：payload=p32(C_addr)+ ‘%012d’ + ‘%6$n’

    举个栗子方便理解：

    比如payload为 “\x8c\x97\x04\x08%2048c%5$n” ，那么我们就可以把 0x0804978c 地址里的内容修改为 0x804 （2048+4字节）。

    再举个栗子：

    例如要把printf的地址 修改为 system地址。我们采取逐字节修改。
    payload这样写：

    printf_got=0x08049778  
system_plt=0x08048320
payload=p32(printf_got)+p32(printf_got+1)+p32(printf_got+2)+p32(printf_got+3)
payload+="%"+str(0x20-16)+"c%5$hhn"  
payload+="%"+str(0x83-0x20)+"c%6$hhn"
payload+="%"+str(0x104-0x83)+"c%7$hhn"
payload+="%"+str(0x08-0x04)+"c%8$hhn"

    %k$hhn表示往偏移k处写入1个字节。
    解读：
    已知我们所有的输入都是从偏移5开始的，所以根据payload，第5个的位置将对应的是printf_got，而因为我们payload前部分已经输出了4个4字节地址了，要想把printf_got的最低字节的数据改为0x20的话，我们还得要输出str(0x20-16)个字节数据。于是我们把前面输出的字节总个数0x20写入了printf_got的最低1字节处，即’\x78’改为了’\x20’。 接下去第6个的位置将对应的是printf_got+1的地址，同样的，我们先前已经输出了0x20，离0x83还有距离，还需要输出str(0x83-0x20)个字节，然后再把0x83写入printf_got+1的地址处，把printf_got的倒数第二低字节的’\x97’改为’\x83’。 以此类推。最终把printf_got的四字节地址数据全部改写为system的地址！
    所以payload展开来表示为： ‘\x78\x97\x04\x08\x79\x97\x04\x08\x7a\x97\x04\x08\x7b\x97\x04\x08%16c%5$hhn%99c
    %6$hhn%129c%7$hhn%4c%8$hhn’

    此外也可以直接利用pwntools的工具fmtstr（详情请百度）：

    fmtstr_payload(offset,{address:data},numbwritten=0,write_size=’size’)

    第一个offset参数是可控的栈偏移（不包括格式化字符串参数），本例子中是第6个参数，所以是5。第三个参数numbwritten指的是在格式化字符串之前就输出了的数据，比如在可控变量前就输出了”Hello”的话，numbwritten就要设置为5。第四个参数选择有 %hnn (byte) , %hn (word) , %n (dword) ，本题我们选择 %hhn所以按照公式，我们的payload可以这样写：

    payload = fmtstr_payload(5, {printf_got:system_plt})

    任意地址写（64位）：

    跟32位的利用差不太多。

    32位程序payload ： p32(泄露地址)+%偏移$x

    64位格式化漏洞例题： （BUUCTF–r2t4）：

    payload=”%64c%9$hn%1510c%10$hnAAA” + p64(stack_chk_fail+2) + p64(stack_chk_fail)

    解读：字符串末尾的AAA纯粹是为了8字节对齐。

    len(“%64c%9$hn%1510c%10$hnAAA”)=24字节

    已知我们输入的内容在偏移6处，那么我们已经输入了24字节内容，再输入内容将会存放在偏移6+3=9，即偏移9处，所以第偏移9我们就用来存放一个指针，这个指针指向stack_chk_fail函数地址+2。由于p64打包又是8字节的，所以偏移10处我们存放指针指向stack_chk_fail函数地址。

    ok，现在payload布局已经解释完了，现在来解释这串字符串。
    %64c%9$hn 表示：打印64个字符，然后%n检测到前面已经打印了64个字符，所以把字符个数64 （0x40）写入到偏移9处指针所指向的地址，即把stack_chk_fail+2修改为 0x0040 ，注意%hn是写入2字节。
    %1510c%10$hn 也同理，再输出1510个，现在是1510+64=1574 ，即0x626，把这两个字节写入到偏移10处指针指向的地址，即把    stack_chk_fail地址修改为 0x0626
    现在__stack_chk_fail函数的地址算是彻底被我们改为了 0x400626 了，这个即后门函数的地址。然后我们故意触发canary保护机制，即能执行我们的后门函数啦！
