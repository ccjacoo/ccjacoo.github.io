    gdb下不同插件

    包含 pwndbg 、peda、gef，它们的功能各不相同，所以需要来回切换。
    安装过程看这篇文章

    https://blog.csdn.net/kevin66654/article/details/86773517

    如果后续需要对堆进行调试分析，需要另外从下面地址安装pwndbg。

    https://github.com/scwuaptx/Pwngdb

    其主要包含有parseheap、heapinfo、chunkinfo 等一些重要的关于heap的指令。

    如何调试程序

    首先得分清：
    gdb ./filename 指的是静态调试
    gdb attach filename 指的是动态调试

    比如对名为pwn的程序进行调试：

    • gdb ./pwn #开启gdb，并且file读取pwn程序
    • b main #对main函数下断点
    • b *0x8048e40 #对某一地址进行下断点
    • b *0x56555000+0x655 #也可以基址+偏移的形式
    • r #运行程序，在输入某次数据后可以Ctrl+c 截断下来，看此时的堆栈情况。
    • start #以汇编的角度运行该程序，汇编代码将呈现在你眼前，另外还有寄存器和栈情况。
      在指令“start”的基础上：
    • n #单步执行不步入直接得结果
    • s #单步并步入函数
    • c #自动运行到下一个断点处
    • got #查看GOT表
    • finish #结束掉（跳过）当前函数，直接就跳到了该函数的leave ret 处
    • q #结束调试,或是输入detach退出

    如何调试exp

    别人也许是用的gdb.attach()自动依附，或者是 raw_input()
    但我个人习惯做法是在exp中想要断点的语句前 加一个 pause() ，然后gdb attach pid

    具体做法是：开两个终端（按shift+ctrl+t 在一个终端的基础上同界面内创建新的终端 ，而shift+ctrl+n 则是在新界面新建终端），假如exp全称为 exp.py，第一个终端输入 gdb exp.py ，然后会自动停在pause()语句处，得到调试程序的PID假如是6845，然后在第二个终端里输入 gdb attach 6845 便可。

    如果有多条pause语句，那么如何执行下一个pause语句？
    第二个终端里先输入c（即continue）等待，然后Alt+1切换至第一个终端，敲一下回车，会显示停在第二个pause处，然后切换回第二个终端，按ctrl+c截断即可继续调试。

    个人经常使用的exp快捷键：

    把选中区域全部注释掉： ctrl + /
    令选中区域缩进： tab
    令选中区域顶格：shift+tab

    等等，要想 gdb attach pid 得注意：
    ubuntu16.04 默认不允许普通用户gdb attach PID调试进程………我原先都是用root权限所以没遇到过这情况。
    解决方法：
    sudo vim /etc/sysctl.d/10-ptrace.conf
    最后一行改为 kernel.yama.ptrace_scope = 0
    去除限制！

    调试指令总结

    详细的gdb调试指令请看：

    https://blog.csdn.net/weixin_30697239/article/details/101103647

    以下列出我个人使用频繁的指令

    gdb-peda下：

    1.查看某一个地址(比如说0xf7e522b0)的详细信息，start，end，offset，位于哪个区域里

    xinfo 0xf7e522b0

    2.查看虚拟映射内存，可以找到libc程序加载的基址，还有heap等其他地址信息

    vmmap

    3.查看地址对应内存内容

    x/10gx 0x80485a6

    4.查看地址对应的汇编代码

    x/20i 0x80484b6

    5.看自栈顶esp开始向下20的栈的内容

    stack 20

    6.查看寄存器ebp-0x20的内容

    x/w $ebp-0x20

    7.查看内容

    telescope 0x8049700

    8.计算我们输入的格式化字符串的地址与这个地址之间的偏移

    fmtarg 0x7fffffffdda8

    9.查看堆分布

    parseheap

    10.查看bins情况

    heapinfoall

    pwndbg下：

    1.system函数在文件中的地址，且当程序没有开启ASLR时，地址不变

    print system

    2.显示所有bin链表

    bins

    3.显示每个堆的数据结构

    heap

    4.查看plt内部函数表

    plt

    5.查看got全局函数表

    got

    6.查看寄存器信息

    regs

    7.查看stack栈

    stack

    8.打印包含返回地址的栈地址

    retaddr

    9.查看某地址的内存，左边是Hex，右边框是字符串

    hexdump 0x7fffd6117fee

    个人感觉pwndbg用起来还是挺舒服的，但我平时常用的还是gdb-peda

    经验：

    1.查找输入数据的去向
    做一道题目时得到的启发：题目要求输入一串字符串，比如我们输入’AAABBBCCC’后ctrl+c断下来之后，
    使用 “find AAABBBCCC” 就可以找得到字符串在内存中所储存的位置啦！

    2.如何应对PIE 地址随机化?
    gdb ./pwn 打开gdb调试 ， 然后 r 加载运行程序 ， 然后在另开一个终端输入 “ps -ef |grep pwn” 得到pwn程序的PID ，然后 “cat /proc/pid/maps” 查看内存信息，得到基址，一般是第一行的start地址，那么实际地址 = 基址+IDA的相对偏移。然后再在第一个终端正在运行的程序里下断点，进行后续操作，比如输入数据后ctrl+c断下来查看内存。
