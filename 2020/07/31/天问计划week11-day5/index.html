    学习：

    组相连映射：把主存块映射到cache固定组的任一行。

    比如说cache有16行，然后2行为一组，所以就有8组。所以取模的时候模 8 即可。

    主存地址有20位，第一部分8位是标记，第二部分3位是组号，第三部分的9位是块内地址。

    举例，如何对0120CH单元进行访问？得到 00000001 001 000001100 ，所以就是第1组中的第001块（即第9块， 9 mod 8=1）中的第12个单元，所以映射到第一组。

    几个概念：

    Hit Rate(命中率)：在cache中的概率

    Hit Time（命中时间）：在cache中的访问时间 （判断时间+访问时间）

    Miss Rate（缺失率）： 1-（Hit Rate）

    Miss Penalty（缺失损失）：访问一个主存块所花时间

    取数据过程：根据主存地址的index索引找到cache中的对应行，然后将tag和对应行的tag进行比对，然后再和V进行与操作。

    淘汰策略问题：（替换算法）

    1.先进先出 FIFO ，命中率不随着组的增大(几行一组)而提高。

    2.最近最少用 LRU （这个更好），算是一种栈算法，最常用的在栈顶，最少用的在栈底，它的命中率随着组的增大而提高。具体实现时，给每个cache行设定一个计数器，根据计数值来记录这些主存块的使用情况，这个计数值称为LRU位。每组4行时，计数器有2位，计数值越小则说明越常用。命中时，被访问行的计数器置0，比其低的计数值都+1，其余不变。未命中且该组未满时，新行计数值置0，其余全+1 。未命中且该组已满时，把计数值为(行数-1)的那一行的主存块淘汰掉，新行计数值置0，其余+1 。

    3.最不经常用 LFU

    4.随机替换 Random

    写策略（cache与主存内容的一致性问题）：

    写操作有两种情况：写命中（Hit），要写的单元已经在cache中；另外就是 写不命中（Miss） 了。

    写命中有两种处理方式：

    1. Write Through：写穿透，直写。同时写cache和主存单元。缺点就是极其的慢！ 解决方法就是使用写缓冲（Write Buffer），当CPU改写cache中数据后会再另外写write buffer写缓冲（也是很快的事），写完后CPU就可以干其他事情了，随后由 Memory Controller 来根据写缓冲，并行地把cache中的值更新到主存单元里，这部分工作是不需要CPU操心的。如果是连续写指令的话，write buffer也是会饱和的（毕竟一般只有4个字大小），容易阻塞。如何解决写缓冲饱和？加一个二级cache

    2. Write Back （一次性回写）：写的时候只修改cache中的内容，而不更新主存单元，只有缺失时才一次性写回。而写操作过程中一般都是锁死主存单元，不让其他设备访问主存单元的数据（因为可能是旧数据了）。而cache中每行有一个标志位（修改位）叫做“dirty-bit”，又称为“脏位”，当这一行被修改过了，则做个标记，等被替换出去后才根据标志位然后修改主存单元内容。

    写不命中有两种处理方式：

    1. 写分配，把主存块装入cache，然后更新相应单元。这个办法试图利用空间局部性，但每次都要从主存中读一个块。
    2. 非写分配，反正在cache没有该数据，所以直接修改主存单元啊，不把主存块装入到cache。

    关于降低缺失率（Miss Rate），cache的大小当然是越大越好（成本也越高），但是Block的size却要适中，一般64字节最好。

    现在的cache也是有多级的，分为一级cache（Level1），二级cache…….

    cache分立：即把数据和指令分开存放在各自的数据和指令cache中。所以就可以做到同时取数据和取指令。一般L1 cache都是分立cache。且L1 cache的命中时间比命中率更重要。

    联合：数据和指令都放在一个cache中。一般L2 cache都是联合cache，且L2 cache的命中率比命中时间更重要。

    首先来认识一下文件：

    traces文件夹中的5个文件都是用于测试csim.c的输入文件。各个文件包含不同指令，用于测试 hits,misses,evictions。

    这些文件中指令的格式如下（举例）：

     I 0400d7d4,8
 M 0421c7f0,4
 L 04f6b868,8
 S 7ff0005c8,8

    可以看出指令的格式为：[空格] 操作 地址，大小

    每行代表一个或两个内存访问。

    操作字段表示存储器的访问类型：

    I 表示指令加载

    L 表示数据加载，只访问cache一次

    S 表示数据存储，只访问cache一次

    M 表示数据修改，意味着先读再写，要访问cache两次

    地址字段指定一个32位的十六进制存储器地址。

    大小字段则表示操作访问的字节数。

    /*
 * Harbin Institute of Technology - Qiuhao Li
 *
 * Anyone can modify this code and then redistribute it to help others.
 *
 * -std=c99
 */

//头文件
#include "cachelab.h"
#include <stdio.h>    /* fopen freopen perror */
#include <stdint.h>    /* uintN_t */
#include <unistd.h> /* getopt */
#include <getopt.h> /* getopt -std=c99 POSIX macros defined in <features.h> prevents <unistd.h> from including <getopt.h>*/
#include <stdlib.h> /* atol exit*/
#include <errno.h>  /* errno */

//宏定义
#define false 0
#define true 1

typedef struct
{
    _Bool valid;    /* flag whether this line/block is valid, zero at first*/
    uint64_t tag;   /* identifier to choose line/block */
    uint64_t time_counter;  /* LRU strategy counter, we should evict the block who has the min time_counter, zero at first */
    /* We don't need to simulate the block, since we just requested to count hit/miss/eviction */
    /*time_counter初始化的时候都是0，其值越大代表这个line最近刚刚被写入——我们不应该替换它——所以valid为0的line的time_counter一定也是0（最小值）,
    因为他们连使用都没有被使用过，即我们一定会先替换valid为0的line*/
}line;
typedef line *entry_of_lines;
typedef entry_of_lines *entry_of_sets;

/*
Data structure:
//首先根据S（set的数目）申请一个数组，该数组元素是lines的入口的指针。接着循环S次每次申请E个line数据结构，并让刚刚的指针数组的元素指向它们

                                                             +-----+
                                                +-----+   +-->Valid|
                                           +---->line0+---+  +-----+
                                           |    +-----+   |
                         +---------------+ |              |  +---+
                         | set0          | |    +-----+   +-->Tag|
                      +--> entry_of_lines+------>line1|   |  +---+
                      |  +---------------+ |    +-----+   |
                      |                    |              |  +-------+
                      |  +---------------+ |    +-----+   +-->Counter|
                      |  | set1          | +---->line2|      +-------+
                      +--> entry_of_lines| |    +-----+
+--------------+      |  +---------------+ |
| cache0       +------+                    |    +-----+
| entry_of_sets|      |  +---------------+ +---->lineX|
+--------------+      |  | set2          |      +-----+
                      +--> entry_of_lines|
                      |  +---------------+
                      |
                      |  +---------------+
                      |  | setX          |
                      +--> entry_of_lines|
                         +---------------+


*/

//将结果设计成结构体，方便函数返回
typedef struct
{
    int hit;
    int miss;
    int eviction;
}result;

//声明函数
entry_of_sets InitializeCache(uint64_t S, uint64_t E);

result HitMissEviction(entry_of_lines search_line, result Result, uint64_t E, uint64_t tag, _Bool verbose);

result ReadAndTest(FILE *tracefile, entry_of_sets cache, uint64_t S, uint64_t E, uint64_t s, uint64_t b, _Bool verbose);

void RealseMemory(entry_of_sets cache, uint64_t S, uint64_t E);



int main(int argc, char * const argv[])
{
    result Result = {0, 0, 0};  //定义返回值
    const char *help_message = "Usage: \"Your complied program\" [-hv] -s <s> -E <E> -b <b> -t <tracefile>\n" \
                         "<s> <E> <b> should all above zero and below 64.\n" \
                         "Complied with std=c99\n";
                         // 加上"\"后可以换行写
    const char *command_options = "hvs:E:b:t:";

    FILE* tracefile = NULL;

    entry_of_sets cache = NULL;

    _Bool verbose = false;    /* flag whether switch to verbose mode, zero for default 可选参数，加上-v后就可以额外打印出一些语句*/
    uint64_t s = 0;    /* number of sets index's bits */
    uint64_t b = 0;    /* number of blocks index's bits */
    uint64_t S = 0; /* number of sets 组的个数*/
    uint64_t E = 0;    /* number of lines 行的个数 */ 

    char ch;    /* command options */

    while((ch = getopt(argc, argv, command_options)) != -1)
    {
        switch(ch)
        {
            case 'h':
            {
                printf("%s", help_message);
                exit(EXIT_SUCCESS);
            }

            case 'v':
            {
                verbose = true;
                break;
            }

            case 's':
            {

                if (atol(optarg) <= 0)    /* We assume that there are at least two sets */
                {
                    printf("%s", help_message);
                    exit(EXIT_FAILURE);
                }
                s = atol(optarg);
                S = 1 << s;
                break;
            }

            case 'E':
            {
                if (atol(optarg) <= 0)  //库函数atol()，即把一个字符串转化为一个整数
                {
                    printf("%s", help_message);
                    exit(EXIT_FAILURE);
                }
                E = atol(optarg);
                break;
            }

            case 'b':
            {
                if (atol(optarg) <= 0)    /* We assume that there are at least two sets */
                {
                    printf("%s", help_message);
                    exit(EXIT_FAILURE);
                }
                b = atol(optarg);
                break;
            }

            case 't':
            {
                if ((tracefile = fopen(optarg, "r")) == NULL)
                {
                    perror("Failed to open tracefile");
                    exit(EXIT_FAILURE);
                }
                break;
            }

            default:
            {
                printf("%s", help_message);
                exit(EXIT_FAILURE);
            }
        }
    }

    if (s == 0 || b ==0 || E == 0 || tracefile == NULL)
    {
        printf("%s", help_message);
        exit(EXIT_FAILURE);
    }

    cache = InitializeCache(S, E);  //初始化cache ,此时 S = 1 << s
    Result = ReadAndTest(tracefile, cache, S, E, s, b, verbose);  //测试文件
    RealseMemory(cache, S, E);   /* Don't forget this in C/C++, and do not double release which causes security problem */
    //printf("hits:%d misses:%d evictions:%d\n", Result.hit, Result.miss, Result.eviction); //释放内存
    printSummary(Result.hit, Result.miss, Result.eviction); //返回统计数据
    return 0;
}

//初始化cache
entry_of_sets InitializeCache(uint64_t S, uint64_t E)
{
    entry_of_sets cache;

    /* use calloc instead of malloc to match the default situation we designed */

    if ((cache = calloc(S, sizeof(entry_of_lines))) == NULL) /* initialize the sets */  //初始化 组
    {
        perror("Failed to calloc entry_of_sets");
        exit(EXIT_FAILURE);
    }

    for(int i = 0; i < S; ++i)  /* initialize the lines in set */  //初始化 组里的每行
    {
        if ((cache[i] = calloc(E, sizeof(line))) == NULL)
        {
            perror("Failed to calloc line in sets");
        }
    }

    return cache;
}

result HitMissEviction(entry_of_lines search_line, result Result, uint64_t E, uint64_t tag, _Bool verbose)
{
    uint64_t oldest_time = UINT64_MAX;
    uint64_t youngest_time = 0;
    uint64_t oldest_block = UINT64_MAX;
    _Bool hit_flag = false;

    for (uint64_t i = 0; i < E; ++ i)
    {
        if (search_line[i].tag == tag && search_line[i].valid) /* tag一致且valid=1 则表示 hit ! */
        {
            if (verbose)    printf("hit\n");  // 加上-v参数就可以打印出这句话
            hit_flag = true;  //用来表示是否hit的标志
            ++Result.hit;
            ++search_line[i].time_counter; /* update the time counter 被使用了，离淘汰又远了一步 */
            break;
        }
    }

    if (!hit_flag)  /* miss */
    {
        if (verbose)    printf("miss");
        ++Result.miss;  //未命中次数+1
        uint64_t i;
        for (i = 0; i < E; ++i)    /* search for the oldest modified block (invalid blocks are oldest as we designed) */
        {
            if (search_line[i].time_counter < oldest_time)
            {
                oldest_time = search_line[i].time_counter;  //更新最少的次数，然后用作下一次比较，还有谁比他更少？！
                oldest_block = i;  //记录更新使用次数最少的块号
            }
            if (search_line[i].time_counter > youngest_time)    /* search for the youngest modified block to update the new block's time counter */
            {
                youngest_time = search_line[i].time_counter; //更新使用最频繁的次数
            }
        }

        search_line[oldest_block].time_counter = youngest_time + 1;
        search_line[oldest_block].tag = tag;

        if (search_line[oldest_block].valid) /* It's a valid block, ++eviction */
        {
            if (verbose)    printf(" and eviction\n");  //valid=0时终于被驱逐，所以语句打印效果是 “miss and eviction”
            ++Result.eviction;  //驱逐次数+1
        }
        else
        {
            if (verbose)    printf("\n");  //语句打印效果是 “miss”  ，即只是miss，还没有到驱逐的地步
            search_line[oldest_block].valid = true;
        }
    }
    return Result;  //返回统计结果
}


result ReadAndTest(FILE *tracefile, entry_of_sets cache, uint64_t S, uint64_t E, uint64_t s, uint64_t b, _Bool verbose)
{
    result Result = {0, 0, 0};
    char ch;
    uint64_t address;
    while((fscanf(tracefile, " %c %lx%*[^\n]", &ch, &address)) == 2)    /* read instruction and address from tracefile and ignore the size */
                                                                        /* address is represented by hexadecimal, use %lx instead of %lu */
    {
        if (ch == 'I')
        {
            continue; /* we don't care about 'I' */
        }
        else
        {
            uint64_t set_index_mask = (1 << s) - 1;
            uint64_t set_index = (address >> b) & set_index_mask;
            uint64_t tag = (address >> b) >> s;
            entry_of_lines search_line = cache[set_index];


            if (ch == 'L' || ch == 'S') /* load/store can cause at most one cache miss */
            {
                if (verbose)    printf("%c %lx ", ch, address);
                Result = HitMissEviction(search_line, Result, E, tag, verbose);
            }

            else if (ch == 'M') /* data modify (M) is treated as a load followed by a store to the same address.
                                   Hence, an M operation can result in two cache hits, or a miss and a hit plus an possible eviction. */
            //M操作指令将两次访问cache
            {
                if (verbose)    printf("%c %lx ", ch, address);
                Result = HitMissEviction(search_line, Result, E, tag, verbose);  /* load, hit/miss(+eviction) */
                Result = HitMissEviction(search_line, Result, E, tag, verbose);  /* store, must hit */
            }

            else    /* ignore other cases */
            {
                continue;
            }
        }
    }
    return Result;
}

//释放内存
void RealseMemory(entry_of_sets cache, uint64_t S, uint64_t E)
{
    for (uint64_t i = 0; i < S; ++i)
    {
        free(cache[i]);
    }
    free(cache);
}

    本题目的是优化矩阵转置。

    最简单的转置实现：

    for (int i = 0; i < N; ++i)
{
    for (int j = 0; j < M; ++j)
    {
        dst[j][i] = src[i][j]

    冲突不命中：实际上就是在访问同一个块中的两个元素的时候，由于中间访问了其它的块，导致已经加载的块被驱逐，进而第二次访问时不命中。

    基于冲突不命中这个问题，要减少miss数，我们可以一次性访问同一个块中的多个元素，访问完以后便不再需要访问这个块了，从而可以大大地减少冲突不命中的数目。假如是8个元素为一个块，那么前8个元素在同一个块中，我们可以直接将这8个元素取出来，然后这8个元素所在的块便不再需要访问了。

    这种方法貌似称为 blocking ，是一种良好的优化技术。以数据块的形式读取数据，就是尽量将一个block读入完全或者写入完全。完全利用完这个block后丢弃，然后读取下一个block，这样防止block利用得不全面，导致冲突不命中的情况发生。

    ​ 关于shell：打印出提示符，然后等待来自stdlin的输入，根据输入执行特定地操作，这样就产生了一种错觉，似乎输入的字符（命令行）控制了程序的执行。

    ​ 命令行是一串ASCII字符，由空格分隔。字符串的第一个参数（argv[0]）是一个可执行程序，或者是shell的内置命令。其余部分是命令的参数。

    ​ 如果第一个参数是内置命令，shell会立即在当前进程中执行。否则shell就会fork新建一个子进程，然后在子进程中执行程序。新建的子进程又叫做作业。通常，作业可以由Unix管道连接的多个子进程组成。

    ​ 如果命令行以＆符号结尾，那么作业将在后台运行，这意味着在打印提示符并等待下一个命令之前，shell不会等待作业终止。 否则，作业在前台运行，这意味着shell在作业终止前不会执行下一条命令行。 因此，在任何时候，最多可以在一个作业中运行在前台。 但是，任意数量的作业可以在后台运行。

    ​ Unix shell支持作业控制的概念，允许用户在前台和后台之间来回移动作业，并更改进程的状态（运行，停止或终止）。在作业运行时，键入ctrl-c会将SIGINT信号传递到前台作业中的每个进程。SIGINT的默认动作是终止进程。类似地，键入ctrl-z会导致SIGTSTP信号传递给所有前台进程。 SIGTSTP的默认操作是停止进程，直到它被SIGCONT信号唤醒为止。

    Unix shell还提供支持作业控制的各种内置命令，例如：

    jobs：列出运行和停止的后台作业
    bg ：将停止的后台作业更改为正在运行的后台作业
    fg ：将停止或运行的后台作业更改为在前台运行
    kill ：终止作业
