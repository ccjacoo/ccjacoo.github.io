    什么是Canary？

    要想Bypass(绕过)stack保护机制，得先明白什么是Canary。

    Canary是用于防止栈溢出被利用的一种方法，原理是在栈的ebp下面放置一个随机数,这个值总是以字节 \x00 结尾,本意是为了保证 Canary 可以截断字符串。保护机制会在函数返回之前会检查这个数有没有被修改，若被改动过则判断发生栈溢出了。

    具体实现如下：

    （32位）：

    我们可以通过IDA工具查看流程图：
    在函数序言部分插入了canary值：

    mov eax,gs:0x14
    mov DWORD PTR [ebp-0xc],eax

    在函数返回之前，会将该值取出，检查是否修改。这个操作即为检测是否发生栈溢出。

    mov eax,DWORD PTR [ebp-0xc]
    xor eax,DWORD PTR gs:0x14
    je 0x80492b2 <vuln+103> # 正常函数返回
    call 0x8049380 <__stack_chk_fail_local> # 调用出错处理函数

    32位下，canary的值存储的位置为 ebp-0x8 ：

        High  
    Address |                 |  
            +-----------------+
            | args            |
            +-----------------+
            | return address  |
            +-----------------+
            | old ebp         |
  ebp =>    +-----------------+
            | ebx             |
ebp-4 =>    +-----------------+
            | unknown         |
ebp-8 =>    +-----------------+
            | canary value    |
ebp-12 =>    +-----------------+
            | 局部变量        |
    Low     |                 |
    Address

    （64位）：

    流程和上面一样，只是汇编代码不同了

    mov rax,QWORD PTR fs:0x28
    mov QWORD PTR [rbp-0x8],rax

    函数返回前进行判断：

    mov rax,QWORD PTR [rbp-0x8]
    xor rax,QWORD PTR fs:0x28
    je 0x401232 <vuln+102> # 正常函数返回
    call 0x401040 <__stack_chk_fail@plt> # 调用出错处理函数

    栈的分布如下：

       High
    Address |                 |
            +-----------------+
            | args            |
            +-----------------+
            | return address  |
            +-----------------+
            | old ebp         |
  rbp =>    +-----------------+
            | canary value    |
rbp-8 =>    +-----------------+
            | 局部变量         |
    Low     |                 |
   Address

    无论是32位还是64位，关于设置Canary的汇编代码都是类似的。

    写在最后

    本篇介绍了什么是Canary以及存储位置，返回函数时检查canary的值有无改变，若值遭修改了，则触发报警，call ___stack_chk_fail 结束程序。

    本系列后续会给出绕过Canary的一些方法，这些方法都是要掌握的。

    方法有：

    • 泄露canary的值
    • one by one 爆破canary的值
    • SSP 攻击
    • TSL攻击
    • 劫持stack_chk_fail 函数
    • 数组下标溢出
