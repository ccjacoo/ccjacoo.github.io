    查看一个ELF文件的信息

    注意，下方程序的文件名为都用filename来代替啦。

    file命令：

    拿到一个文件时，首先执行 file 命令：

    file filename

    判断它是ELF文件还是EXE文件。判断它是32位程序还是64位程序。判断它是静态编译还是动态编译。

    结果如下：

    filename: ELF 32-bit LSB executable, Intel 80386, version 1 (SYSV), dynamically linked, interpreter /lib/ld-, for GNU/Linux 2.6.32, BuildID[sha1]=0ffa519ef269240388d07b316b1ef863216e24cb, not stripped

    其中我们只关注这几个关键字：
    32-bit表明它是32位程序。
    dynamically linked 表示它是动态编译的，链接了一些函数库。
    GNU/Linux 2.6.32 表示libc版本

    checksec命令：

    checksec filename

    显示该程序保护机制的开启情况，只有事先知道了本题的保护机制，我们才能制定应对措施，理清利用思路，不盲目。

    结果如下：

    • Arch: i386-32-little
    • RELRO: Partial RELRO
    • Stack: No canary found
    • NX: NX enabled
    • PIE: No PIE (0x8048000)

    关于保护机制的详细信息请看笔记《ELF保护机制》.

    ldd命令：

    这个命令查看程序使用的共享库文件，找到libc库的名字。

    ldd filename

    结果如下(我的是ubuntu 16.04)：

    linux-gate.so.1 =>  (0xf7fa1000)

    libc.so.6 => /lib/i386-linux-gnu/libc.so.6 (0xf7dc9000)
    /lib/ld-linux.so.2 (0xf7fa3000)

    ROPgadget命令：

    当需要构造ROP链攻击时，需要找到类似 pop rdi；ret 这样的gadgets代码片段。所以需要使用到ROPgadget工具。

    基本格式如下：

    ROPgadget –binary filename

    上面的命令可以寻找到所有的gadgets。
    结果如下（我只列举一部分，因为太多啦）：

    ………..
    0x00000000004006ab : leave ; ret
    …………
    0x0000000000400590 : pop rbp ; ret
    0x0000000000400733 : pop rdi ; ret
    0x0000000000400731 : pop rsi ; pop r15 ; ret
    …………

    当然我们还可以寻找的更详细一点，定向搜寻：

    搜寻寄存器 （比如搜寻所有关于rdi寄存器的gadgets）：

    ROPgadget –binary filename –only ‘pop|ret’ | grep “rdi”

    寻找字符串：

    ROPgadget –binary pwn –string “/bin/sh”

    寻找系统调用：

    ROPGadget –binary pwn | grep “int 0x80”

    寻找bss段：

    readelf -S pwn | grep “.bss”

    注意这里有一个小知识点：每一个gadget都有一个ret用于返回，ret指令取栈顶的数据作为下次跳转的位置，即 pop eip ： eip=[esp]; esp=esp+4;

    one_gadgets命令：

    one_gadget libc库版本

    结果如下：

    $ one_gadget /lib/x86_64-linux-gnu/libc.so.6

    0x45216 execve(“/bin/sh”, rsp+0x30, environ)
    constraints:
    rax == NULL

    0x4526a execve(“/bin/sh”, rsp+0x30, environ)
    constraints:
    [rsp+0x30] == NULL

    0xf02a4 execve(“/bin/sh”, rsp+0x50, environ)
    constraints:
    [rsp+0x50] == NULL

    0xf1147 execve(“/bin/sh”, rsp+0x70, environ)
    constraints:
    [rsp+0x70] == NULL

    有时候程序里不好构造ROP链时可以考虑使用one_gadget。
    假如我们选择偏移为 0xf02a40，前提是知道了libc基址，所以one_gadget的内存地址为 libc基址+0xf02a4
    但要注意的是：当程序没有 libc 时，是无法使用one_gadget 进行RCE的！

    one_gadget本质是：

    execve(“/bin/sh”)

    readelf命令：

    该命令可以获取elf头信息
    dynstr(动态链接的字符串)中储存了函数名，而节的地址可以用 readelf -S filename 来查看，其中节区是REL类型的则包含重定位表项。
    查看各段和节区的信息包括地址：

    readelf -S filename

    其中可寻找bss段：

    readelf -S pwn | grep “.bss”

    关于readelf的参数还有许许多多，自行百度叭。
