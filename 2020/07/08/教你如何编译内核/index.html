    教你如何编译内核

    今天翻阅笔记时发现我在2020年4月29日前后学习有关 kernel pwn 的有关知识，我又重新整理回顾了一遍。算是为以后调试 kernel pwn 打好基础吧。

    本篇将介绍如何编译内核。

    关于编译内核的过程我主要参考了钞sir师傅的视频：
    https://www.bilibili.com/video/BV1KE411c7mu
    以及网路上一些编译内核文章作为辅助参考：
    https://blog.csdn.net/weixin_30326741/article/details/96114177

    编译内核过程中有踩过一些坑，待会在下文也分享出来。

    0.小前提：

    我自己用的是ubuntu 18.04 ,然后虚拟机开机后显示的是普通用户账户，注意我们要用 root 用户进行登录，点击“未列出”后输入root和密码。
    但是在此之前需要设置一个root账户（默认是没有root账户的）之后才能用root用户登录。
    具体设置root账户的办法请百度叭。

    1.编译内核：

    1.1 下载内核源码：

    要编译内核源码之后才可以进行Linux内核调试
    官网下载源码进行编译: https://www.kernel.org/
    我下载了最新版的 5.6.7
    我把所有源码文件都放在了 /usr/src/ 目录下

    安装依赖：

    sudo apt-get install git fakeroot build-essential ncurses-dev xz-utils libssl-dev bc

    1.2 相关设置：

    make menuconfig

    注意下面的配置:
    KernelHacking –>
    选中Compile the kernel with debug info
    选中Compile the kernel with frame pointers
    去掉 Write protect kernel read-only data structures
    在Processor type and features —>
    去掉Paravirtualized guest support

    1.3 编译内核：

    然后最后编译：

    make -j8

    make modules_install

    make install

    之后进入编译过程。
    这段过程将十分的漫长！由于我自己初次实践的时候配置搞错了，所以曾编译了3次内核！对，这么长的编译时间我经历了3次，呜呜，我太菜了。

    编译完之后就可以从 ./arch/x86/boot/ 中拿到 bzlmage 镜像 ，从源码根目录中拿到 vmlinux 文件。

    2.编译 busybox –构建文件系统：

    2.1 下载最新版的busybox的源码：

    wget https://busybox.net/downloads/busybox-1.31.0.tar.bz2

    2.2 相关设置：

    make menuconfig

    注意以下配置：
    打开busybox Setting —-> Build Options -> Build Busybox as a static binary 编译成为静态文件
    关闭以下选项：
    Linux System Utilities -> Support mounting NFS files system //网络文件系统
    Networking Utilities –> inetd //internet超级服务器

    2.3 编译安装：

    make install

    安装完后ls看见非常多的文件。
    继续配置：

    cd _install

    mkdir proc sys dev etc etc/init.d

    vim etc/init.d/rcS

    在该文件中写入以下内容：

    #!/bin/sh
mount -t proc none /proc
mount -t sysfs none /sys
/sbin/mdev -s

    然后给予权限：

    chmod +x etc/init.d/rcS

    2.4 创建文件系统：

    在 _install目录下执行：

    find . | cpio -o –format=newc > ../../rootfs.img

    对指令的解读：找到当前目录下全部文件，并进行打包，制作成cpio，形成一个根文件系统镜像放到父目录的rootfs.img里。
    以后我们自己调试的时候好像也会经常使用到该类似指令。

    我找了好久的资料，才发现这个是在内核启动时候去读取的。所以我们可以自定义，事先往里面放我们想要的文件（比如run.sh或脏牛代码）的话就相当于是初始化了吧（只是个人理解，可能不准确），然后编译出一份rootfs.img 镜像文件 。这样的话我们下一次再qemu打开内核时就不会是啥都没有的默认空内核了，多了一些我们添加进去的文件。了。
    操作：先把/usr/src/目录下旧的rootfs.img删掉，然后进到 busybox文件系统的 _install目录下 继续使用上面 find 和 cpio 指令进行编译，然后就会生成一个新的rootfs.img 镜像文件。

    3.安装 qemu-system 模拟器：

    3.1 安装qemu-system：

    sudo apt-get install qemu-system

    然后建议自己去官网找gdb的源码然后编译，以免后续使用过程中会出什么问题。不建议使用自带的原有的gdb
    http://ftp.gnu.org/gnu/gdb/

    依次输入下列指令：

    tar xjf gdb-7.8.tar.gz
    cd gdb-7.8/
    ./configure –target=arm-linux
    make
    make install

    接下来就可以使用我们的 qemu 模拟器来运行刚刚编译好的内核了：

    3.2 qemu运行内核：

    用qemu运行内核，建议写一个sh脚本，这个脚本就是自定义我们的内核文件、镜像、文件系统、安全措施，开启smap、smep等安全措施，然后还可以为gdb开启一个端口，方便我们后续调试remote连接。
    我创建的sh脚本名为 start.sh

    sh脚本内容为：

    qemu-system-x86_64 \
-kernel ./bzlmage \
-initrd ./rootfs.img \
-append "console=ttyS0 root=/dev/ram rdinit=/sbin/init" \
-cpu kvm64,+smep,+smap \
-nographic \
-gdb tcp::6666

    到这一步算是都设置完了。

    然后执行sh脚本：

    ./start.sh

    终端会闪过一大堆数据后停住，然后 敲回车 即可出现 # 提示符。

    内核运行成功！

    总结：

    现在这些文件是比较重要的：
    1.从 ./arch/x86/boot/ 中拿到 bzlmage 镜像
    2.可不断手动更新内容的文件系统镜像文件：rootfs.img
    3.在源码根目录下生成的 vmlinux
    4.内核启动脚本：start.sh
