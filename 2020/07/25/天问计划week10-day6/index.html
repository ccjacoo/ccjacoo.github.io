    根据以上我已经大致了解了程序的运行原理。

    首先会call调用 read_six_numbers函数，读取6个数字，且最后返回的时候 rsp 的位置存着第1个数字的地址。然后后面的数字依次存入 rsp+0x4 ，rsp+0x8，rsp+0xC , rsp+0x10 , rsp+0x14 中

    然后第一个数必须是1，要不然会引爆炸弹。

    然后跳转到 0x400f30处， 把 rsp+0x4 处的值给rbx，把 rsp+0x18 处的值给rbp

    ok，现在rbx里存的是第2个数字， rbp存的是第6个数字之后+4的地址。

    然后跳转回0x400f17开始执行核心代码了：

    把rbx-0x4 赋给eax，eax=(rsp+0x4 )-0x4 ，所以现在eax 中存的是 第1个数字。

    add eax,eax ，即eax=eax*2

    然后比较eax和rbx。就是比较 rsp与 rsp+0x4 ，即第1个数字和第2个数字

    也就是说第2个数字将会是第1个数字的2倍。ok，找到关键点了。

    绕过爆炸函数后接着把 rbx+0x4，即 (rsp+0x4)+0x4= rsp+0x8 ，开始轮到第3个数字了。

    然后比较 rbx和rbp 。rbx肯定还没到达rbp的地址，所以返回0x400f17处继续循环。

    所以根据以上，就可以大致猜出，第1个数字是1，且后一个数字是前一个数字的两倍。

    所以最后得到这6个数字为 ： 1 , 2 , 4 , 8 , 16 , 32

    3.phase_3:

    知识点：jb 和 ja 用无符号整数的比较。指令 ja 表示 CF=0 且ZF=0 即A>B转移； 指令 jb 表示 CF=1 且ZF=0 即A <B转移；

    而 jg 则表示 带符号整数的大小比较。 jg则表示大于关系跳转。

    看到了调用了 sscanf，所以这个函数前面一定会有一个字符串常量存储需要读取的数据格式，所以字符串常量一定是 0x4025cf ，用gdb打印出来确定格式：

    gdb-peda$ p (char*)(0x4025cf)
    $1 = 0x4025cf “%d %d”

    我靠，学到了。 一个字符串常量来存储需要的数据格式。

    可以看到数据格式”%d %d”，故需要输入两个整数。而 eax存储的是scanf函数的返回值。

    由开头的语句得知：

    lea rcx,[rsp+0xc]

    lea rdx,[rsp+0x8]

    所以知道了第1个数存在rsp+0x8处，第2个数存在rsp+0xc处。

    代码段的后半段有很多的jmp，且都是通过0x400f75处的处理结果来判断跳转的，所以推测0x400f75处是switch语句。

    0x400f75处的语句是根据 rax*8+0x402470 的结果跳转的。 而rax就是我们输入的第一个数。

    查看 “基址”：0x402470：

    gdb-peda$ x/10xg 0x402470
0x402470:    0x0000000000400f7c    0x0000000000400fb9
0x402480:    0x0000000000400f83    0x0000000000400f8a
0x402490:    0x0000000000400f91    0x0000000000400f98

    可以发现，若eax=1时，会跳转到 [0x402470+1*8]= [0x402478] = 0x400fb9 ，即跳转到对应的case语句。

    而每一个case语句一开始都会将某个参数赋给eax，然后统一跳转到 0x400fbe 处。而在这个地方则是将我们输入的第二个数和每个case传给eax中不同的值进行比较，相等就跳过炸弹否则爆炸。

    而刚才分析了eax的值是根据第一个值跳转到不同的case得到的。那么有多少个case就应该有多少个解题的答案，我们只需要确定第一个数然后顺着跳转到其中一个case中，然后看这个case中的常量值是多少即为我们输入的第二个值了。

    所以根据之前分析的，我第一个数输入1，然后顺着跳转来到了0x400fb9处，找到了第二个数为0x137，即十进制的311

    所以最后我只要输入 1 , 311 即可通过本关。（当然，本题还有其他的选择，eax取0,1,2,3,4,5,6都可）

    4.phase_4：

    jbe指令：（无符号整数）低于(below)或等于(equal)则跳转

    jle指令：（有符号整数）小于(less)或等于(equal)则跳转

    jge指令：（有符号整数）大于(large)或等于(equal)则跳转

    根据x86汇编的约定， rdi，rsi，rdx 分别存储第1,2,3个参数，rax存储函数的返回值

    在fun_4的函数体中发现它又有调用自身，于是想到了这里用的是递归。

    关于本题的调试手法：

    set args string.txt 把答案集作为参数传入，自动输入。

    然后 b *0x401048 在调用fun_4处下断点。

    输入r开始运行，前面三个phase自动输入答案跳过，然后在第4个phase处输入 3 0 ，然后程序自动截断，断在了0x401048处。

    此时 rdi是我们输入的第1个数0x3，rsi=0x0，rdx=0xe

    然后 si 步入fun_4函数。

    运行几步后ecx=0x7，然后就把ecx和edi(我们输入的第1个数)进行比较。

    比较结果：我们输入的3比7小，所以不跳转，继续顺序往下执行，只见把edx的值减1后继续调用fun_4函数。

    所以这里我们如果令输入的第1个数为7，会怎样？

    重新来一遍，这一回输入 7 0，然后第一遍比较的时候由于 符合“第1个数大于等于0x7”所以跳转到 0x400ff2，

    而0x400ff2处的代码将检查 是否符合“第1个数小于等于0x7”。

    哈哈，这回明白了，要想同时符合这两个条件，第1个数就只能为 7 了 。

    又由于fun_4函数返回时有一个判断，第2个数必须等于0才不会引爆炸弹。

    所以综上，得到本题的答案： 7 0

    另外附上某位师傅逆向出的fun_4的C源程序：（我是没能力逆向出源程序的，只是大致了解程序的功能，wtcl）

    int func4(int target, int step, int limit) {
  /*由调用约定得： edi = target; esi = step; edx = limit */
  int temp = (limit - step) * 0.5;  //右移1位，即除以2
  int mid = temp + step;
  if (mid > target) {
    limit = mid - 1;
    int ret1 = func4(target, step, limit);
    return 2 * ret1;
  } else {
    if (mid >= target) {
      return 0;
    } else {
      step = mid + 1;
      int ret2 = func4(target, step, limit);
      return (2 * ret2 + 1);
    }
  }
}
